# Skill 标准模板

> 构建新Skill时使用的标准结构

---

## 基本信息

```yaml
---
name: [skill-name]
description: |
  [一句话描述这个Skill做什么]
  触发场景：[什么时候使用]
  核心能力：[这个Skill能提供什么]
---
```

## 核心理念（可选）

> 如果新Skill有独特的哲学基础或理念，在此描述

## 核心框架

```markdown
【核心框架】

框架名称：[名称]
核心结构：
  - 要素A：[描述]
  - 要素B：[描述]
  - 要素C：[描述]
关系描述：[各要素如何关联]
框架原理：[为什么这个框架有效]
```

## 方法论

```markdown
【方法论】

名称：[名称]
适用条件：[什么时候使用]
核心步骤：
  1. [第一步]
  2. [第二步]
  3. [第三步]
关键原则：[必须遵循的原则]
常见陷阱：[容易犯的错误]
变体：[针对不同情况的调整]
```

## 心智模型

```markdown
【心智模型】

模型名称：[名称]
核心洞察：[一句话描述核心洞察]
适用场景：[什么时候使用这个模型]
使用方式：[如何应用]
局限：[模型的局限性]
```

## 决策启发式

```markdown
【决策启发式】

启发式名称：[名称]
决策规则：[如果X，则Y]
适用条件：[在什么情况下有效]
失效条件：[什么时候可能出错]
```

## 理论基础

```markdown
【理论基础】

来源领域：[来自哪个领域]
核心概念：[关键概念定义]
支撑理论：[相关理论支撑]
实证基础：[是否有实践验证]
```

## 底层哲学

```markdown
【底层哲学】

核心命题：[最根本的主张]
支撑假设：
  - 假设A：[描述]
  - 假设B：[描述]
认识论：[如何看待认知]
方法论：[如何看待行动]
价值观：[核心价值]
边界：[适用边界]
```

## 使用指南

### 触发条件

> 什么时候激活这个Skill

### 标准流程

> 结构化的使用步骤

### 变体方案

> 针对不同情况的调整

### 质量检查

> 使用后如何验证效果

---

## 模板使用说明

构建新Skill时：

1. **读取模板** — 先读取本文件了解标准结构
2. **填充内容** — 按模板结构填写各部分
3. **保持精简** — 删除不适用的部分，不添加新章节
4. **确保可执行** — 方法论和流程必须具体可操作
5. **标注边界** — 诚实说明Skill的局限
