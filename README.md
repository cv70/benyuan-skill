## 核心理念

本源不是分析表面现象，而是**从一切信息中提取根本源头，构建可用的本质框架**。

当向本源提出请求时，会得到：
- **核心框架**：隐藏现象背后的结构模型
- **方法论**：可复用的操作步骤和原则
- **底层哲学**：支撑这一切的第一性原理
- **道之本源**：最究竟的规律、本体、本源
- **完整Skill**：基于本质提取构建的可用工具

## 工作流程

```
用户内容
    ↓
第一步：内容解构 — 理解用户提供了什么
    ↓
第二步：深度研究 — 补充内容盲区，建立全局视野
    ↓
第三步：溯源追问 — 追溯本质，追问至少3层
    ↓
第四步：深度提取 — 萃取核心框架/方法论/哲学
    ↓
第五步：Skill构建 — 按skill-template.md生成完整Skill
```

### 第一步：内容解构

解构用户内容，识别核心概念、观点、隐藏假设、方法路径、边界条件。

### 第二步：深度研究

从六个维度研究：领域定义、历史脉络、存在根基、方法体系、边界条件、关联图谱。

### 第三步：溯源追问

从时间/空间/逻辑/认知四个维度追问，至少追问3层。

### 第四步：深度提取

萃取三类本质：
- **核心框架**：隐藏现象背后的结构模型
- **核心方法论**：可复用的操作原则
- **底层哲学**：支撑第一性原理的假设和信念

### 第五步：Skill构建

按 `references/skill-template.md` 标准结构生成完整、可执行的Skill。

## 输出示例

**核心框架**：表达=信息×接收者×渠道

**核心方法论**：定题→构思→起草→修改→定稿
