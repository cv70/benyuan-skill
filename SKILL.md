## 第一步：内容解构

### 解构模板

```
【内容解构】

核心概念：["反复出现或定义的关键术语"]

- [ ] 是否明确了适用边界？

## 第二步：深度研究

对主题进行全方位研究，补充用户内容的盲区。

### 研究维度

| 维度 | 研究问题 | 输出 |
|------|---------|------|
| 领域定义 | 这个领域的核心定义是什么？与相似领域有何区别？ | 精准定义 |
| 历史脉络 | 这个领域如何起源？如何演变至今？关键转折点是什么？ | 发展路径 |
| 存在根基 | 这个领域/概念为什么存在？解决什么问题？ | 根本原因 |
| 方法体系 | 核心方法和技法学什么？有哪些主流流派？ | 方法地图 |
| 边界条件 | 适用边界在哪里？什么情况下会失效？ | 边界标注 |
| 关联图谱 | 与哪些邻近领域相关？起源自哪些领域？ | 关联网络 |

### 研究原则

- **追溯第一手资料**：不止于二手解读，优先查找原始论文/著作
- **多角度审视**：不盲从单一视角，记录争议之处
- **诚实记录**：明确标注不确定和推测的部分
- **填补盲区**：特别关注用户内容未覆盖的维度

### 研究记录格式

```
【深度研究】

领域定义：
  定义：[精准定义]
  边界：[与其他领域的区分]

历史脉络：
  起源：[何时何地起源]
  关键节点：[列出3-5个关键转折]
  演变趋势：[向什么方向演进]

存在根基：
  核心问题：[领域要解决什么]
  根本价值：[为什么重要]

方法地图：
  主要流派：[列出主流方法]
  核心技巧：[关键技术]

边界条件：
  适用：[什么情况下有效]
  失效：[什么情况下可能出错]

关联网络：
  上游起源：[来自哪些领域]
  下游应用：[影响到哪些领域]
  相似领域：[相关但不同的领域]
```

## 第三步：溯源追问

对研究和用户内容进行溯源，追问至少3层。

### 溯源四维度

| 维度 | 追问句式 | 输出 |
|------|---------|------|
| 时间 | 「这个从哪里来？」「最初为什么出现？」「更早之前呢？」 | 历史起源 |
| 空间 | 「它是由什么构成的？」「最小组分是什么？」「组分从何而来？」 | 最小粒子 |
| 逻辑 | 「它成立的前提是什么？」「前提的前提呢？」「最终前提是什么？」 | 第一性原理 |
| 认知 | 「这个认知从哪里来？」「谁定义的？」「为什么这样定义？」 | 心智本源 |

### 溯源记录格式

```
【溯源追问】

时间维度：
  第1层：[来源/起源]
  第2层：[更早的来源]
  第3层：[最初的源头]
  终点：[触及本源]

空间维度：
  第1层：[构成组分]
  第2层：[组分的组分]
  第3层：[最小单元]
  终点：[万物本源]

逻辑维度：
  第1层：[直接前提]
  第2层：[前提的前提]
  第3层：[最终公理]
  终点：[第一性原理]

认知维度：
  第1层：[认知来源]
  第2层：[更早的来源]
  第3层：[心智根源]
  终点：[心智本源]
```

## 第四步：深度提取

这是本Skill的核心能力。

### 4.1 提取核心框架（Core Framework）

识别并提取隐藏在内容背后的结构模型。

**核心问法**：
- 「这个内容/领域背后隐藏的结构是什么？」
- 「如果简化到最简结构，是什么？」
- 「这个结构可以解释哪些现象？」
- 「有哪些看似不同但结构相同的事物？」

**提取模板**：

```
【核心框架】

框架名称：[给框架起一个精准、独特的名字]
核心结构：
  - 要素A：[描述这个要素]
  - 要素B：[描述这个要素]
  - 要素C：[描述这个要素]
  （根据实际情况，要素可以是2-5个）
关系描述：[各要素如何关联，形成什么结构]
框架原理：[为什么这个框架有效/存在]
适用范围：[可以解释哪些现象]
局限：[无法解释什么]
```

### 4.2 提取核心方法论（Core Methodology）

从具体方法中萃取可复用的方法论。

**核心问法**：
- 「这个方法背后的通用原则是什么？」
- 「这个步骤能否简化到更普适的形式？」
- 「如果有变量，什么是恒定的？」
- 「这个方法的核心公式是什么？」

**提取模板**：

```
【核心方法论】

方法论名称：[命名]
核心洞察：[一句话描述为什么这个方法有效]

适用条件：
  - 适合：[什么情况下使用]
  - 不适合：[什么情况下不要用]

核心步骤：
  1. [第一步：要做什么，注意什么]
  2. [第二步：要做什么，注意什么]
  3. [第三步：要做什么，注意什么]

关键原则：
  - 原则A：[描述]
  - 原则B：[描述]

常见陷阱：
  - 陷阱A：[描述] → 应对：[如何避免]
  - 陷阱B：[描述] → 应对：[如何避免]

变体版本：
  - 变体A：[针对情况A的调整]
  - 变体B：[针对情况B的调整]
```

### 4.3 提取底层哲学（Root Philosophy）

提取支撑方法的底层哲学。

**核心问法**：
- 「这个方法背后的信仰/信念是什么？」
- 「如果否认这个假设，会怎样？」
- 「这个方法和哪种世界观相连？」
- 「这个领域的实践者默认什么价值观？」

**提取模板**：

```
【底层哲学】

核心命题：[最根本的主张，一句话]

支撑假设：
  - 假设A：[描述] → 如果否认这个：[会怎样]
  - 假设B：[描述] → 如果否认这个：[会怎样]

认识论：
  - 这个领域如何看待认知和知识？
  - 什么是可知？什么是不可知？

方法论：
  - 这个领域如何看待行动和实践？
  - 什么是对的做事方式？

价值观：
  - 核心价值取向是什么？
  - 什么是好的，什么是坏的？

边界：
  - 这些假设的适用边界在哪里？
  - 什么时候这些假设可能失效？
```

## 第五步：构建Skill

读取 `references/skill-template.md` 获取标准结构。

### 构建流程

1. **读取模板** — 先读取skill-template.md了解标准结构
2. **映射内容** — 将第四步提取的内容对号入座
3. **补充缺失** — 如果有缺失环节，回到前面的步骤补充
4. **验证完整性** — 检查是否有遗漏的关键部分

### 生成Skill输出格式

```
【Skill生成】

名称：[精准独特的名字]
定位：[一句话描述解决什么问题]

核心框架：[来自第四步]

方法论：[来自第四步]

心智模型：[如果有的话，简洁描述]

决策启发式：[如果有的话，简洁描述]

理论基础：[来自研究阶段]

底层哲学：[来自第四步]

使用指南：
  触发条件：[什么时候用]
  标准流程：[详细步骤]
  变体方案：[不同情况如何调整]

边界说明：

## 表达DNA

### 句式特征
- **层层剥茧**：从现象到本质，逐步深入
- **先破后立**：解构常见认知再揭示根本
- **终始推演**：从本质出发演绎应用
- **留有余地**：本源之外保持敬畏

### 词汇使用
- 高频词：本源、框架、方法论、底层、萃取、追溯、涌现、源于,归于
- 独用语：归于、源自、生成、构成、本体、萃取、解构、溯源
- 慎用语：绝对真理、唯一原因（用「可能」「或许」替代）

### 节奏控制
- 理解阶段：细致、全面、不遗漏
- 研究阶段：深入、多角度、填补盲区
- 提取阶段：精炼、直击本质、结构清晰
- 构建阶段：完整、可操作、符合模板

---

## 诚实边界

- **研究局限**：研究深度受限于可获取的资料质量和数量
- **认知边界**：部分领域可能触及人类认知边界，无法进一步追问
- **文化局限**：本源框架本身来自人类认知，可能不适用于所有智慧形态
- **时效性**：方法和框架可能随时间演进而失效
- **主观性**：提取过程涉及主观判断，可能存在偏差
